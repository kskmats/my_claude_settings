# MCP設定同期

`~/.claude.json`（実際の設定）からリポジトリ内の`.claude.json`（テンプレート）へ片方向同期する。
機密情報は自動でプレースホルダーにマスキングされる。

## ワークフロー

### 1. 同期スクリプトを実行

```bash
python3 ~/.claude/scripts/sync-mcp-config.py
```

出力はJSON形式で `applied`（適用されたプレースホルダー一覧）、`warnings`（警告）、`servers_synced`（同期されたサーバー一覧）を含む。

### 2. 警告を確認

出力の `warnings` が空でない場合、ユーザーに警告内容を報告し、対応を相談する。

### 3. README.md を更新

出力の `applied` に基づいて、README.md の「マスキングされている値」テーブルを更新する。

テーブル形式:
```markdown
| プレースホルダ | 説明 |
|--------------|------|
| `<PLACEHOLDER>` | 日本語の説明 |
```

- `applied` に含まれるプレースホルダーを全て記載する
- 説明は日本語で簡潔に記述する（例: `<Slack User Token>` -> `Slack のユーザートークン`）
- 既存のテーブルを完全に置き換える

### 4. 変更内容を確認

```bash
git diff
```

差分をユーザーに表示し、内容に問題がないか確認する。
特に以下の点をチェック:
- APIキーやトークンの実値が残っていないこと
- 個人パスが残っていないこと
- プレースホルダーが適切に設定されていること
