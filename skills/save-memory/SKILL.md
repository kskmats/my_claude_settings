# ナレッジ蓄積スキル

セッションで得た技術的知見・トラブルシューティング・再利用可能な手順をナレッジベースに蓄積する。

## ナレッジベースの構成

全ナレッジを `~/.claude/skills/` 配下にグローバル保存する（プロジェクトのgitを汚さないため）。

```text
~/.claude/skills/mem-{topic}/
  SKILL.md        # サマリテーブル + 概要
  references/     # コード例・詳細手順
```

プロジェクト固有の知見はテーブルのソース列に `(プロジェクト名固有)` と記載して区別する。

## 実行手順

### 1. セッションの学びを抽出

- 今回のセッションで解決した問題、得た知見、再利用可能な手順を整理
- プロジェクトの CLAUDE.md やドキュメントも参照

### 2. 対象カテゴリを決定

既存の `mem-{topic}` ディレクトリを確認:

```bash
ls -d ~/.claude/skills/mem-* 2>/dev/null
```

- 該当するカテゴリがあれば既存ファイルに追記
- なければ新規 `mem-{topic}` ディレクトリを作成

topic名は英小文字・ハイフン区切り（例: `aws`, `terraform`, `kubernetes`, `claude-code`, `troubleshooting`）

### 3. SKILL.md を更新

各 `mem-{topic}/SKILL.md` は以下の構造:

```markdown
---
name: mem-{topic}
description: {topic}に関する技術ナレッジ。過去のセッションで蓄積した知見・ベストプラクティス・トラブルシューティング情報。
---

# {Topic} ナレッジ

| 項目 | 内容 | ソース |
|------|------|--------|
| エントリ1 | 簡潔な説明 | 確認元（実体験/ドキュメント等） |
| エントリ2 | 簡潔な説明 | 確認元 |

## 補足

必要に応じて補足説明を記載。
詳細な手順やコード例は `references/` を参照。
```

### 4. references/ にコード例・詳細手順を格納

SKILL.md のテーブルだけでは伝わらない詳細は `references/` に分離:

- `references/{topic-detail}.md` の形式で保存
- SKILL.md から `references/` 内のファイルを参照リンクで言及する

### 5. 重複チェックとマージ

既存エントリと内容が重複する場合:

- 同じ項目があれば内容を更新・統合
- 古い情報は最新の知見で上書き
- テーブル行の追加ではなく、既存行の更新を優先

### 6. 更新結果を報告

以下の形式で報告:

```markdown
## 保存したナレッジ

| カテゴリ   | 追加/更新 | 内容                              |
|------------|-----------|-----------------------------------|
| mem-aws    | 追加      | ECS Blue/Greenデプロイの注意点    |
```

## 注意事項

- APIキー・シークレット・固有のリソースIDは含めない
- 他のセッション・プロジェクトでも再利用できる汎用的な形式で記述
- SKILL.md は簡潔に保ち、詳細は references/ に分離
- 1つのテーブル行は1行で完結させる（セル内改行しない）
